## 系统要求

### 运行环境
- **操作系统**: Windows 10/11 (64位)
- **Python**: 3.9 或更高版本
- **内存**: 最低 4GB，推荐 8GB 或以上
- **硬盘**: 至少 500MB 可用空间

### 必需依赖
- PyQt5 >= 5.15.9
- pydicom >= 2.4.4
- cryptography >= 41.0.0
- numpy >= 1.24.0
- 其他依赖详见 requirements.txt

## 安装说明

### 方法1: 从源码运行

```bash
# 1. 克隆或下载项目
cd dicom-tool

# 2. 安装依赖
pip install -r requirements.txt

# 3. 运行程序
cd src
python main.py
```

### 方法2: 使用打包版本

1. 运行 `build.bat` 进行打包
2. 打包完成后，在 `dist` 目录找到可执行文件
3. 双击运行 `DICOM图像数据处理工具.exe`

## 快速开始

### 1. 打开DICOM文件

**方式1**: 点击菜单 `文件 > 打开` 或按 `Ctrl+O`  
**方式2**: 点击工具栏的 `打开` 按钮

选择 `.dcm` 格式的 DICOM 文件即可加载。

### 2. 查看图像

- **缩放**: 使用鼠标滚轮或点击 `放大`/`缩小` 按钮
- **平移**: 按住鼠标左键拖动
- **窗宽窗位**: 拖动滑块或输入数值调节
- **适应窗口**: 点击 `适应窗口` 按钮自动缩放

### 3. 查看和编辑标签

- 在右侧 `DICOM标签` 标签页查看所有标签
- 使用搜索框快速定位标签
- 双击标签值进行编辑
- 选择标签后点击 `应用` 保存修改

### 4. 敏感信息脱敏

**步骤**:
1. 点击菜单 `工具 > 敏感信息脱敏` 或工具栏 `脱敏` 按钮
2. 选择脱敏级别:
   - **级别1**: 基础脱敏（患者姓名、ID、出生日期）
   - **级别2**: 标准脱敏（符合HIPAA）
   - **级别3**: 严格脱敏（完全匿名化）
   - **级别4**: 加密脱敏（RSA加密保护）
3. 如选择级别4，可选择公钥或生成新密钥对
4. 点击 `确定` 执行脱敏

### 5. 数据压缩

**步骤**:
1. 点击菜单 `工具 > 数据压缩` 或工具栏 `压缩` 按钮
2. 选择压缩级别:
   - **级别1**: 无损压缩（诊断用途）
   - **级别2**: 标准压缩（临床审查）
   - **级别3**: ROI压缩（数据存档）
3. 调整压缩质量（级别2和3）
4. 如选择级别3，可在图像查看器中绘制ROI区域
5. 点击 `确定` 执行压缩

### 6. 保存文件

- **保存**: `文件 > 保存` 或 `Ctrl+S`
- **另存为**: `文件 > 另存为...`

## 功能详解

### 图像查看器

#### 窗宽窗位调节
- **窗位(Window Center)**: 控制显示的中心亮度值
- **窗宽(Window Width)**: 控制显示的对比度范围
- **自动**: 自动计算最佳窗宽窗位
- **重置**: 恢复默认设置

#### ROI绘制（用于压缩级别3）
1. 确保已选择压缩级别3
2. 在图像查看器中按住鼠标左键拖动绘制矩形
3. 释放鼠标完成一个ROI
4. 可绘制多个ROI
5. 在压缩对话框中管理ROI

### 标签编辑器

#### 功能
- **搜索**: 快速定位标签
- **过滤**: 按标签组过滤（患者信息、检查信息、图像信息）
- **编辑**: 双击或选择后修改值
- **删除**: 删除不需要的标签
- **导出**: 导出所有标签到文本文件

#### 敏感标签标识
敏感标签会用红色背景标识，包括：
- 患者姓名
- 患者ID
- 出生日期
- 地址
- 电话等

### 敏感信息脱敏

#### 级别详解

**级别1 - 基础脱敏**
- 处理字段: 患者姓名、ID、出生日期
- 适用: 基本隐私保护
- 可逆性: 不可逆

**级别2 - 标准脱敏**
- 处理字段: 基础字段 + 地址、电话、机构、科室、日期等
- 符合: HIPAA标准
- 可逆性: 不可逆

**级别3 - 严格脱敏**
- 处理字段: 所有可识别信息
- 特殊处理: 所有UID重新生成、日期偏移
- 可逆性: 不可逆
- 适用: 数据共享和研究

**级别4 - 加密脱敏**
- 处理字段: 标准脱敏涉及的所有字段
- 加密方式: RSA非对称加密
- 可逆性: 可逆（需要私钥）
- 注意: 妥善保管私钥！

#### 使用自定义公钥
1. 准备PEM格式的公钥文件
2. 在脱敏对话框中点击 `浏览...`
3. 选择公钥文件
4. 执行脱敏

#### 生成新密钥对
1. 在脱敏对话框中点击 `生成新密钥对`
2. 选择保存目录
3. 系统会生成 `public_key.pem` 和 `private_key.pem`
4. **重要**: 私钥丢失将无法解密数据！

### 数据压缩

#### 压缩级别详解

**级别1 - 无损压缩**
- 格式: JPEG Lossless 或 JPEG 2000 Lossless
- 压缩比: 2-4x
- 质量: 100%（无损）
- 适用: 诊断用途

**级别2 - 标准压缩**
- 格式: JPEG Baseline 或 JPEG 2000
- 压缩比: 8-15x
- 质量: 90%+
- 适用: 临床审查、日常诊断

**级别3 - ROI压缩**
- 格式: JPEG 2000 有损压缩
- 压缩比: 20-40x
- 特性: ROI区域高质量、非ROI区域高压缩
- 适用: 数据存档、远程会诊

#### ROI管理
- **绘制**: 在图像查看器中拖动鼠标绘制矩形
- **清除**: 清除所有ROI
- **保存**: 保存ROI到JSON文件
- **加载**: 从文件加载已保存的ROI
- **设置**: 分别设置ROI和非ROI区域的压缩质量

## 常见问题

### Q1: 无法加载DICOM文件
**原因**: 
- 文件格式不支持
- 文件损坏
- 缺少解码器

**解决方法**:
- 确认文件是标准DICOM格式（.dcm）
- 尝试用其他DICOM查看器打开验证
- 安装额外的解码器: `pip install pylibjpeg pylibjpeg-libjpeg gdcm`

### Q2: 图像显示为空白或全黑
**原因**: 窗宽窗位设置不当

**解决方法**:
- 点击 `自动` 按钮自动计算窗宽窗位
- 手动调整窗宽窗位滑块
- 点击 `重置` 恢复默认设置

### Q3: 脱敏后如何恢复原始数据？
**情况1**: 使用级别1-3脱敏
- **无法恢复**，这些级别是不可逆的

**情况2**: 使用级别4加密脱敏
- 需要对应的私钥
- 使用解密工具恢复

**建议**: 脱敏前备份原始文件

### Q4: 压缩后图像质量下降
**原因**: 选择了有损压缩（级别2或3）

**解决方法**:
- 如需高质量，选择级别1无损压缩
- 调高压缩质量参数
- 使用ROI压缩保护关键区域

### Q5: ROI压缩时没有保护效果
**可能原因**:
- 未绘制ROI区域
- 使用了不支持ROI的压缩格式

**解决方法**:
- 确认已在图像查看器中绘制ROI
- 使用级别3压缩（支持ROI）
- 检查ROI是否正确保存和加载

### Q6: 加密脱敏时密钥管理
**重要提示**:
- 私钥丢失将**无法**恢复加密数据
- 建议将私钥保存在安全位置
- 可以备份私钥到多个位置
- 公钥可以公开分享

### Q7: 程序运行缓慢
**可能原因**:
- DICOM文件过大
- 内存不足
- 同时加载多个大文件

**解决方法**:
- 关闭其他程序释放内存
- 使用ROI功能减少处理区域
- 压缩后再处理

### Q8: 如何批量处理多个文件？
当前版本暂不支持批量处理，需要逐个处理文件。

**替代方案**:
- 可以保存ROI配置复用
- 使用相同的脱敏配置
- 记录处理步骤后批量重放
